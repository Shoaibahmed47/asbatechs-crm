Asbatech CRM — Attendance Security Audit

Document: Employee & Admin attendance security review
Product: Asbatech CRM · Module: Time & Attendance
Generated: June 5, 2026

Summary
Core employee monitoring (tab close, cursor idle, breaks, clock in/out) is implemented and working. Laptop sleep requires the desktop agent. Two API routes lack authentication (critical). One logic bug may false-alarm when switching browser tabs while Attendance stays open in background.

5. What is working well

  • Tab close vs tab switch (main business rule)
  • Cursor idle detection on Attendance page
  • Break rules (official + extra with reason)
  • Server-side away compliance and break minutes
  • Admin employee detail with live totals
  • Agent health date scoping (today, not yesterday)
  • Desktop agent code for sleep/lock (when installed)
  • API auth on most attendance routes

6. Errors and gaps

Critical Security holes

High Logic / trust issues

IDProblemEffect
1Tab switch + cursor idle conflictIf Attendance tab stays open in background, cursor idle may fire after ~10s even though switching tabs is allowed.
2Laptop sleep without agentSleep/lock not detected in browser only. Employees without verified agent show Sleep: 0.
3Monitoring only on /attendanceOther CRM pages are not browser-monitored; agent needed for background.
4Clock-in agent check is UI-onlyAPI allows clock-in without agent verification.
5“Manager notified” textUI says manager; server notifies admins only.

Medium Incomplete features

  • AdminAttendanceLivePanel built but not wired to dashboard
  • Daily/range report tables not fully shown in UI
  • No notification bell in header for attendance alerts
  • Managers have APIs but no full attendance dashboard like admin
  • Re-clock-in same day may not reset all tab/cursor counters

8. Recommended fix priority

  1. Add authentication to attendance-summary and exports/attendance APIs
  2. Pause cursor idle when Attendance tab is hidden (fix tab-switch false alarm)
  3. Enforce agent check on clock-in API (not only UI)
  4. Fix copy — “Admin notified” instead of “manager notified”
  5. Add sleep status badge on employee Attendance page
  6. Wire notification UI for admins
