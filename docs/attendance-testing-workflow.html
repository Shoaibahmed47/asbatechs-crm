2. Employee — daily workflow (happy path)

  1. Log in at /login with employee credentials.
  2. Open sidebar → Attendance (employees only; admins use Dashboard).
  3. Install / verify agent (first time or if status not Running):
    • Click Install Agent → copy PowerShell command → run as Administrator → enter CRM password.
    • Click Verify Agent → badge should show Running.
  4. Clock in (Mon–Fri only; weekend blocked):
    • Click Clock in.
    • Expected: Live status = Active (working); Clock in time recorded; work timer starts.
    • If late (after grace): late minutes stored; explanation popup rules apply (see §5).
  5. Start break (once per day):
    • Click Start break → popup: choose type (Lunch, Prayer, Tea/coffee, Personal, Other) + optional note.
    • Expected: Status = Break; Start break disabled; End break enabled.
  6. End break:
    • Click End break → popup: describe where you went / what you did (required, min 3 chars).
    • Expected: Status = Active; break minutes added to Today’s summary; admin gets notification with times + reason.
  7. Clock out at end of shift:
    • Click Clock out.
    • Expected: Shift closed; work hours finalized; if early leave → explanation rules apply (see §5).
  8. Review Today’s summary (clock in/out, work hours, break minutes) and Breaks table at bottom.

4. Employee — explanation popups (late / early leave / absence)

Popup timing
Production: popup on the next working day (weekends skipped).
Development: popup can appear same day (yellow “Test mode” banner).
Rule: Popup cannot be dismissed without submitting reason → clock-in blocked until done.

4.1 Late arrival

  1. Clock in after expected time + grace (e.g. 7:16 with 7:00 start and 15 min grace).
  2. On next working day, open Attendance → Late arrival explanation modal appears.
  3. Submit reason (min 3 chars) → popup closes → clock-in allowed.
  4. Skip reason → popup stays → clock-in blocked.

Example: Friday late → popup Monday (not Saturday/Sunday).

4.2 Early leave

  1. Clock out before shift end time.
  2. Next working day → Early leave explanation modal.
  3. Submit reason → clock-in allowed. No grace on early leave.

4.3 Absence (did not come)

  1. Employee absent on a working day (no clock-in record).
  2. Next working day when they open Attendance → Absence explanation modal for that date.
  3. Submit reason → clock-in allowed. Admin sees reason in Absence column + notification.

Example: Monday present, Tuesday absent, Wednesday return → Wednesday popup for Tuesday.

4.4 Absent day (no popup same day)

Absence does not trigger a popup on the absent day itself. Admin report shows Absent for that date.

6. Admin — daily workflow

  1. Log in as admin → open Executive Dashboard (/dashboard).
  2. Review Agent health table: employee name, check-in time, agent state, late, early leave, absence, last seen, shift status.
  3. Click Late / Early leave / Absence badges to view employee reasons and times.
  4. Click a row → Employee attendance detail panel (timeline, breaks, export).
  5. Edit employee Check-in time via Edit button in table (per-employee schedule modal).
  6. Open Admin Control or dashboard office section → set Office hours (expected check-in, shift end, late grace minutes).
  7. Check Notifications for: away alerts, tab close, break ended (type, times, reasons), absence explanations.
  8. Open Attendance report (/attendance/report) for date filters, daily/range reports, exports.

8.3 Edge cases

  • Friday late → Monday: No popup Sat/Sun; popup Monday.
  • Friday absent → Monday: Absence popup Monday for Friday.
  • Pending late + try clock in: Blocked until reason submitted.
  • On time clock-in: No late popup.
  • Within grace: e.g. 7:14 for 7:00 + 15 grace = not late.
  • Logout / login: Session + attendance state persist for the day.
